## Estrutura de Erros

Todos os erros seguem o mesmo formato:

```json
{
  "ok": false,
  "error": {
    "code": "CÓDIGO_DO_ERRO",
    "message": "Descrição legível do erro em português."
  }
}
```

## Códigos HTTP

| Status | Significado |
|--------|-------------|
| `200` | Sucesso |
| `201` | Criado com sucesso (nova ativação) |
| `400` | Requisição inválida (body malformado ou campos faltando) |
| `401` | Não autorizado (token inválido, expirado ou revogado) |
| `402` | Saldo insuficiente |
| `404` | Recurso não encontrado |
| `409` | Conflito (ex: tentar cancelar ativação já concluída) |
| `429` | Rate limit atingido |
| `502` | Erro na operadora de SMS |
| `503` | Sistema em manutenção |

## Tabela Completa de Códigos de Erro

| Código | HTTP | Causa | Solução |
|--------|------|-------|---------|
| `UNAUTHORIZED` | 401 | Token inválido, revogado ou ausente | Verifique seu token no painel |
| `BAD_REQUEST` | 400 | Body JSON inválido ou campos obrigatórios ausentes | Verifique os campos `serviceId` e `country` |
| `NOT_FOUND` | 404 | Ativação ou recurso não encontrado | Verifique se o ID pertence à sua conta |
| `INSUFFICIENT_BALANCE` | 402 | Saldo menor que o custo do número | Carregue saldo no painel |
| `NO_NUMBERS_AVAILABLE` | 409 | Nenhum número disponível neste país/serviço agora | Tente outro país ou aguarde alguns minutos |
| `ACTIVATION_LIMIT` | 429 | Limite de ativações simultâneas atingido | Conclua ou cancele ativações existentes |
| `MAINTENANCE` | 503 | Operadora em manutenção | Aguarde e tente novamente |
| `SERVICE_NOT_FOUND` | 404 | `serviceId` inválido | Consulte `/api/v1/services` para IDs válidos |
| `CANNOT_CANCEL` | 409 | Tentativa de cancelar após SMS recebido | Não é possível cancelar — o SMS já chegou |
| `ALREADY_FINISHED` | 409 | Ativação já expirou ou foi cancelada | Verifique o status antes de cancelar |
| `RATE_LIMITED` | 429 | 60 req/min excedido | Aguarde 60 segundos e tente novamente |
| `PROVIDER_ERROR` | 502 | Erro interno da operadora | Tente novamente em alguns segundos |
| `CANCEL_ERROR` | 502 | Falha ao comunicar cancelamento à operadora | Tente novamente; o saldo será estornado se possível |

## Tratamento de Erros no Código

<CodeGroup>
```python Python
import requests

def api(method, path, **kwargs):
    r = requests.request(method, f"{BASE_URL}{path}", headers=headers, **kwargs)
    body = r.json()
    
    if not body.get("ok"):
        code = body["error"]["code"]
        msg = body["error"]["message"]
        
        if code == "UNAUTHORIZED":
            raise Exception(f"Token inválido. Verifique seu API Key: {msg}")
        elif code == "INSUFFICIENT_BALANCE":
            raise Exception(f"Saldo insuficiente: {msg}")
        elif code == "NO_NUMBERS_AVAILABLE":
            raise Exception(f"Sem números disponíveis: {msg}")
        elif code == "RATE_LIMITED":
            import time; time.sleep(60)
            return api(method, path, **kwargs)  # Retry
        else:
            raise Exception(f"Erro API [{code}]: {msg}")
    
    return body["data"]
```

```javascript Node.js
async function api(method, path, body) {
  const res = await fetch(`${BASE_URL}${path}`, {
    method, headers,
    body: body ? JSON.stringify(body) : undefined
  });
  
  const json = await res.json();
  
  if (!json.ok) {
    const { code, message } = json.error;
    
    switch (code) {
      case "UNAUTHORIZED":
        throw new Error(`Token inválido: ${message}`);
      case "INSUFFICIENT_BALANCE":
        throw new Error(`Saldo insuficiente: ${message}`);
      case "NO_NUMBERS_AVAILABLE":
        throw new Error(`Sem números: ${message}`);
      case "RATE_LIMITED":
        await new Promise(r => setTimeout(r, 60000));
        return api(method, path, body); // Retry
      default:
        throw new Error(`[${code}] ${message}`);
    }
  }
  
  return json.data;
}
```
</CodeGroup>

## Erros de Rede vs. Erros de API

<Note>
  **Erros de rede** (timeout, sem conexão) são diferentes dos **erros de API**. Para erros de rede, sempre implemente retry com backoff exponencial, pois o Convex backend pode estar respondendo ao request internamente mesmo que você não receba a resposta.
</Note>

```python
import requests, time

def safe_api_call(fn, max_retries=3):
    """Retry automático para erros de rede (não para erros de API como 402, 404)."""
    RETRYABLE_STATUS = {429, 500, 502, 503, 504}
    
    for attempt in range(max_retries):
        try:
            r = fn()
            if r.status_code not in RETRYABLE_STATUS:
                return r
            # Backoff exponencial apenas para erros temporários
            time.sleep(2 ** attempt)
        except requests.exceptions.ConnectionError:
            time.sleep(2 ** attempt)
    
    raise Exception("Todas as tentativas falharam")
```
